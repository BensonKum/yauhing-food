## 原因分析：為何「倉存及取貨未正常」？

### 主因（90% 機會）
**Ben 用緊 `yauhing-food.com` 自訂網域，而該網域的 SSL 證書 Firebase 仲未批到**

Firebase 自動 SSL 證書流程：
1. DNS A record 指向 `199.36.158.100` ✅ 已完成
2. Firebase 偵測 DNS 變更 → 自動申請 Let's Encrypt 證書
3. 證書頒發 + 部署（**需 5-30 分鐘，有時更長**）

**問題徵兆：**
- Chrome 顯示 `NET::ERR_CERT_COMMON_NAME_INVALID`
- 連線會用「不安全」標籤
- 點「進階」→「繼續前往」可能可以訪問，但用戶通常會停喺呢個畫面

### 次因（10% 機會）
**舊版 Service Worker cache 殘留**

雖然已 bump 到 v5 + skipWaiting()，但部分用戶的瀏覽器可能：
- SW 註冊失敗（404 SW 路徑變更）
- 已有舊 SW（v1-v4）卡住，新 SW 無法接管
- PWA 已安裝到主畫面，使用離線模式

## 如果 1 小時後 SSL 仲未得

### 排錯步驟
1. **檢查 DNS 是否完全傳播**
   - `nslookup yauhing-food.com 8.8.8.8` 應返回 `199.36.158.100`
   - `nslookup yauhing-food.com` (本機 DNS) 應一致

2. **手動重新申請 Firebase 證書**
   - Firebase Console → Hosting → yauhing-food.com → 「重新連接」

3. **檢查 GoDaddy 是否有舊 Forwarding 殘留**
   - 之前設過 301 Forwarding，雖然用戶確認已刪，但要再 double-check
   - GoDaddy 舊版 `mya.godaddy.com` → Domains → yauhing-food.com → DNS

4. **檢查是否有 CAA record 阻擋**
   ```
   dig yauhing-food.com CAA
   ```
   應返回 `0 issue "letsencrypt.org"` 或 `0 issuewild "letsencrypt.org"`
