# 產品更新工作指引

> 每次做產品相關工作之前，必須睇完呢份文件先郁手！  
> 最後更新：2026-07-10

---

## ⚠️ 必須遵守的步驟

### 1. 接收指令時 — 確認範圍

收到「產品改名/新增/停售」指令時，**必ず先確認**：

- 邊個 SKU？
- 改乜？（名稱？價格？圖片？狀態？）
- **唔改嘅部分係咪真係唔使郁？**（主動問清楚）
- 括號内容係咪要保留？（例如「玉米鮮肉餃 (8隻裝)」— 括號内係規格描述，除非特別說明，唔郁）

**問清楚先郁，唔好估！**

---

### 2. 執行時 — 同步更新所有位置

產品資料涉及多個地方，改完一個要檢查其餘全部：

| 位置 | 何時郁 | 負責系統 |
|------|--------|---------|
| `products.json` | 幾乎每次都郁 | 倉存系統 + 前台 |
| `index.html` | 名稱/圖片/顯示必郁 | 前台網站 |
| Firestore `inventory` | 名稱/庫存必郁 | 倉存系統實時資料 |
| Firestore `products` | 停售/新增必郁 | 前台購物車 |
| BOGO 優惠列表 | 產品喺優惠内必郁 | 前台優惠功能 |
| Service Worker | 任何資料變更要郁 | PWA 緩存 |

---

### 3. 完成後 — 驗證清單

每改完一樣，檢查以下各項：

- [ ] `products.json` — JSON 語法正確（用 `node -e "JSON.parse(require('./products.json'))"` 驗證）
- [ ] `index.html` — 卡片名稱、alt 文字、圖片路徑正確
- [ ] Firestore — `inventory` doc 名稱同步更新，庫存數字正常顯示
- [ ] BOGO 列表 — 如有，確保名稱同步
- [ ] Console — 確認冇 Error

---

### 4. JSON 語法錯誤預防

**最重要！**

每次改完 `products.json`，**立即用 node 驗證**：

```powershell
node -e "JSON.parse(require('./products.json'))"
```

如果出現 `SyntaxError`：
- 檢查結尾有冇 `]`
- 檢查有冇漏 `,` 或 多餘 `,`
- 確認冇 `trailing comma`（最後一項後面唔可以有 `,`）

**驗證通過先 commit，唔好跳過！**

---

### 5. 提交時 — 統一 commit，一次搞掂

唔好分開幾個 commit，改完所有地方之後一次過 commit。

Commit message 格式：`[產品] YHxxx 更新 — 內容簡述`

---

### 6. 萬一出錯 — 立即 revert，唔好繼續補救

如果發現改錯咗：
1. **立即** `git revert` 或 `git reset --hard` 
2. 回復正常之後再重新做
3. 唔好喺錯嘅基礎上繼續補，越整越乱
