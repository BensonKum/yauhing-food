每次改任何產品之前，**跟呢個順序檢查**：

- [ ] **Excel** 有呢個 SKU？名稱係乜？
- [ ] **products_v2.json** 有呢個 SKU？
- [ ] 兩邊名稱一致？
- [ ] Firestore 有呢個 doc？
- [ ] 涉及多個 SKU（例如 A/非A）？**全部都查晒**
- [ ] 改完要唔要同步 Firestore？
- [ ] 需要新增圖片？

| 改動類型 | 要改邊個 |

| 改價格 | index.html 卡片 + products_v2.json + Firestore doc |

## ✅ 正確流程（每次新增 SKU）

```
1. 搵到 Excel 入面嘅 SKU 定義
2. 確認 products_v2.json 係咪已有
   → 已有：檢查名稱/價格是否一致
   → 冇有：新增到 products_v2.json
3. 如果係新 SKU（唔係舊嘥變體）：
   → 新增 index.html 卡片（或用現有卡加強）
4. Firestore 同步（doc 以 name 為 ID）
5. Commit + Push + Firebase Deploy
6. 通知 Ben 測試
```

## 🚨 常見錯誤模式

| 錯誤 | 後果 | 點避免 |
|------|------|--------|
| 淨係查 A 版 SKU | 非 A 版 SKU 完全漏空 | 查 SKU 時睇埋相鄰號碼 |
| 只改 index.html | JSON 顯示錯，發票匹配失敗 | 雙軌同步，兩邊一齊改 |
| Firestore 未同步 | 員工庫存系統睇唔到 | 每個 SKU 改完立即更新 Firestore |
| 忘記備份 | 改錯冇得回滾 | 郁手前先備份 products_v2.json |
